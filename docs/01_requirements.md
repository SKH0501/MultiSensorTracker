# 요구사항 정의서

## 1. 문서 목적

본 문서는 C++ 기반 다중 센서 융합 객체 추적 및 경보 시뮬레이터의 기능 요구사항과 비기능 요구사항을 정의한다.

본 프로젝트는 실제 센서 장비 없이 CSV 형태의 입력 데이터를 사용하여 RADAR/EO 센서 데이터를 처리하고, 좌표 변환, 센서 융합, 객체 추적, 위험구역 판단 과정을 소프트웨어로 재현하는 것을 목표로 한다.

---

## 2. 프로젝트 개요

### 2.1 프로젝트명

C++ 기반 다중 센서 융합 객체 추적 및 경보 시뮬레이터

### 2.2 프로젝트 목적

본 프로젝트의 목적은 실제 운용 환경을 직접 구성하기 어려운 상황을 소프트웨어 입력 데이터로 재현하고, 센서 데이터 처리 및 상태 판단 로직을 검증하는 것이다.

이를 통해 다음 역량을 학습하고자 한다.

* C++ 기반 객체지향 설계 및 구현
* CSV 기반 센서 데이터 입력 처리
* RADAR 거리/방위각 데이터의 좌표 변환
* RADAR/EO 센서 데이터 융합
* 객체별 상태 추적
* 위험구역 접근 여부 판단
* 시험 케이스 기반 검증

---

## 3. 입력 데이터 정의

시스템은 CSV 파일을 통해 센서 데이터를 입력받는다.

### 3.1 입력 데이터 형식

```txt
time,sensor_type,target_id,value1,value2
0,RADAR,T1,100,30
0,EO,T1,85,50
1,RADAR,T1,105,32
1,EO,T1,89,52
```

### 3.2 필드 정의

| 필드명         | 설명                         | 예시        |
| ----------- | -------------------------- | --------- |
| time        | 센서 데이터가 수집된 시간             | 0, 1, 2   |
| sensor_type | 센서 종류                      | RADAR, EO |
| target_id   | 추적 대상 객체 ID                | T1, T2    |
| value1      | RADAR의 경우 거리, EO의 경우 x 좌표  | 100       |
| value2      | RADAR의 경우 방위각, EO의 경우 y 좌표 | 30        |

## 4. 기능 요구사항

| ID        | 요구사항        | 설명                                                             |
| --------- | ----------- | -------------------------------------------------------------- |
| REQ-F-001 | 센서 데이터 입력   | 시스템은 CSV 파일에서 센서 데이터를 읽을 수 있어야 한다.                             |
| REQ-F-002 | 센서 타입 구분    | 시스템은 RADAR 데이터와 EO 데이터를 구분하여 처리해야 한다.                          |
| REQ-F-003 | RADAR 좌표 변환 | 시스템은 RADAR의 거리와 방위각 데이터를 x, y 좌표로 변환해야 한다.                     |
| REQ-F-004 | EO 좌표 처리    | 시스템은 EO 센서의 x, y 좌표 데이터를 처리할 수 있어야 한다.                         |
| REQ-F-005 | 센서 데이터 매칭   | 시스템은 동일 시간, 동일 target_id를 기준으로 RADAR 데이터와 EO 데이터를 매칭해야 한다.     |
| REQ-F-006 | 센서 융합       | 시스템은 RADAR 좌표와 EO 좌표를 가중 평균 방식으로 융합해야 한다.                      |
| REQ-F-007 | 객체 상태 추적    | 시스템은 target_id별 이전 위치와 현재 위치를 관리해야 한다.                         |
| REQ-F-008 | 속도 계산       | 시스템은 이전 위치와 현재 위치를 기반으로 객체의 이동 속도를 계산해야 한다.                    |
| REQ-F-009 | 위험구역 판단     | 시스템은 객체가 감시구역 또는 위험구역에 진입했는지 판단해야 한다.                          |
| REQ-F-010 | 센서 누락 판단    | 시스템은 특정 시간에 필요한 센서 데이터가 누락된 경우 SENSOR_LOST 상태로 판단해야 한다.        |
| REQ-F-011 | 상태 출력       | 시스템은 객체 상태를 NORMAL, WATCH, WARNING, SENSOR_LOST 중 하나로 출력해야 한다. |
| REQ-F-012 | 결과 로그 출력    | 시스템은 시간별 객체 위치, 속도, 상태 판단 결과를 콘솔 로그로 출력해야 한다.                  |

---

## 5. 상태 정의

| 상태          | 의미                                  |
| ----------- | ----------------------------------- |
| NORMAL      | 객체가 일반 영역에 위치한 상태                   |
| WATCH       | 객체가 감시구역에 접근한 상태                    |
| WARNING     | 객체가 위험구역에 진입했거나 속도 기준을 초과한 상태       |
| SENSOR_LOST | 필요한 센서 데이터가 누락되어 정상적인 융합 판단이 어려운 상태 |
